If left blank, the jobs are in alphabetical order. But if you would like to have another jobs order, then you may define an ordinal set to get an advanced order control The entries in the ordinal are separated by "<" character.
Each match of the ordinal set in a job's full name, would virtually insert the ordinal number in front of the matching entry, when comparing job names.

Example

Precondition:

  • Jobs:
    • HelloApp/CI_Pipeline
    • HelloApp/Deploy_to_Test
    • DemoApp/Deploy_to_Test
    • DemoApp/Deploy_to_Prod
  • Ordinal Set:
    • CI_Pipeline < Deploy_to_Test < Deploy_to_Prod

      • I.e. string "CI_Pipeline" has ordinal number "#0", "Deploy_to_Test" has "#1", and "Deploy_to_Prod" has "#2"
Therefore the following is compared:
  • HelloApp/#0_CI_Pipeline
  • HelloApp/#1_Deploy_to_Test
  • DemoApp/#1_Deploy_to_Test
  • DemoApp/#2_Deploy_to_Prod
Therefore this results in job order:
  1. DemoApp/Deploy_to_Test
  2. DemoApp/Deploy_to_Prod
  3. HelloApp/CI_Pipeline
  4. HelloApp/Deploy_to_Test
